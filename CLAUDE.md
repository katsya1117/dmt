# d社マニュアルメンテナンスツール

Adobe AIR製デスクトップアプリの代替Webアプリ。
サービスマニュアルのリソースファイルを型式別フォルダへアップロードするツール。

## アーキテクチャ

```
[ブラウザ: React + TypeScript]
        ↕ HTTP
[Express (Node.js): 運用PCで常駐]
  ・Sambaマウントフォルダへのファイルアクセス
  ・顧客PHPサーバーへのアップロード中継
  ・将来：処理A/B（VB/Python）の呼び出し
        ↕ REST multipart POST
[顧客PHPサーバー]
  ・型式別フォルダにファイルを保存
```

## ドメイン知識

- **型式**: 車種 × 年式の組み合わせ（約700種類）。アップロードの基本単位
- **型式別DB**: SQLiteファイル。型式ごとのリソース定義。これを忘れると差し替えが反映されない
- **処理A**: VB製。XMLを分割し型式別DBを生成（Windows依存）
- **処理B**: Python製。変更XMLをソートしPDF/SWFに変換（Windows + Adobe依存）

## リソースファイル種別

設定ファイルで管理する（コードに直書きしない）:

| 種別 | 形式 | 用途 |
|------|------|------|
| ベースXML | .xml | HTMLマニュアル動的生成ソース |
| マニュアル画像化 | .pdf / .swf | マニュアルの画像表示 |
| 図版データ | .swf / .svg | マニュアル内の図・イラスト |
| キャプション | .xml | 図版のキャプション定義 |
| 型式別DB | .db | リソース・付随情報マップ（SQLite） |

## APIエンドポイント（Express）

| メソッド | パス | 説明 |
|----------|------|------|
| GET | /api/katashiki | 型式一覧 |
| GET | /api/katashiki/:id/files | 型式のアップロード対象ファイル一覧 |
| POST | /api/upload/:id | 顧客PHPへ転送 |
| POST | /api/jobs/process-a | 処理A呼び出し（将来実装） |
| POST | /api/jobs/process-b | 処理B呼び出し（将来実装） |

## 設計方針

- ReactはExpressとだけ通信する（PHP・処理A/Bを意識しない）
- ファイル種別は設定で管理し、SWF廃止等の変更に備える
- 処理A/BのAPIは口だけ作り501を返す（将来の統合への布石）
- Electronは不採用（頻繁なバージョンアップ・将来の多拠点化に非対応）

# ユースケース記述 — アカウント認証機能

（作成: 2026-07-12。`01_ユースケース図.md`のユースケースに対応。現在の実装を基に記述）

## UC-A02: アカウントを新規追加する

| 項目 | 内容 |
|---|---|
| アクター | 社内メンテナンス担当者 |
| 目的 | 新規のアカウント認証情報（ユーザー名・パスワード等）を1件登録する |
| 事前条件 | 一覧画面を開いている |
| 事後条件（成功） | 新しいレコードがDBに追加され、一覧に反映される |
| 事後条件（失敗） | 何も変更されない。エラー内容がフォーム上に表示される |

**正常処理**
1. 「新規追加」ボタンを押す
2. 入力フォームダイアログが開く（ユーザー名・パスワードは必須、その他の項目は任意）
3. 必要事項を入力し「追加」を押す
4. サーバーに送信され、成功するとダイアログが閉じ「追加しました」というトーストが表示される
5. 一覧が自動的に再取得され、新しい行が表示される

**例外処理**
- ユーザー名またはパスワードが未入力 → 送信前にクライアント側検証で当該項目に赤字エラーを表示（サーバーに送信されない）
- 既に存在するユーザー名を指定 → サーバーが409を返し、ユーザー名欄に「このユーザー名は既に使われています」と表示
- その他の通信エラー等 → ダイアログ上部に汎用エラーメッセージを表示

**関連ユースケース**: なし

## UC-A03: アカウント情報を編集する（UC-A04削除／UC-A05リストアを含む）

| 項目 | 内容 |
|---|---|
| アクター | 社内メンテナンス担当者 |
| 目的 | 既存アカウントの情報を修正する。「削除フラグ」の操作により論理削除・リストアも同じ画面で行う |
| 事前条件 | 一覧に編集対象のアカウントが表示されている |
| 事後条件（成功） | 該当レコードが更新される。削除フラグをONにすれば論理削除（UC-A04）、OFFに戻せばリストア（UC-A05）となる |
| 事後条件（失敗） | 何も変更されない |

**正常処理**
1. 対象行の「編集」アイコンを押す
2. 編集フォームダイアログが開き、既存の値が初期表示される
3. 項目を修正する。解約させたい場合は「削除フラグ」スイッチをONに、リストアしたい場合はOFFに戻す
4. 「更新」を押す
5. 成功すると「更新しました」というトーストが表示され、一覧に反映される。削除済みの行は一覧から消えず、「状態」列が「削除済み」表示になる（半透明表示）

**例外処理**
- 必須項目（ユーザー名・パスワード）を空にした → 赤字エラー表示
- ユーザー名を他の既存アカウントと重複する値に変更 → 409エラー、ユーザー名欄にエラー表示
- その他の通信エラー等 → ダイアログ上部に汎用エラーメッセージを表示

**関連ユースケース**: UC-A04（削除する）・UC-A05（リストアする）は本ユースケースの`<<extend>>`（削除フラグの値による特殊系。UI・APIともに専用の操作は無い）

## UC-A06: Excelを取り込み一括反映する（UC-A07差分プレビューを含む）

| 項目 | 内容 |
|---|---|
| アクター | 社内メンテナンス担当者 |
| 目的 | 得意先から支給されるExcel台帳（全件のマスタファイル、または変更分だけを手動で切り出した差分ファイル）を取り込み、複数件の追加・変更・削除・リストアを一括で反映する |
| 事前条件 | 取り込み対象のExcelファイル（.xlsx/.xls）が手元にある |
| 事後条件（成功） | 承認した差分の内容がDBに反映される。ファイルにあっても承認しなかった行、およびファイルに存在しない既存の行は一切変更されない |
| 事後条件（失敗） | 何も変更されない |

**正常処理**
1. 「Excel取り込み（差分プレビュー）」ボタンを押し、ファイルを選択する
2. ファイルがサーバーへアップロードされ、サーバー側で現在のDBとの差分が計算される
3. 差分プレビューダイアログが開き、区分（追加／変更／削除／リストア）ごとに変更内容がテーブル表示される。ユーザー名・パスワード・削除フラグの変更は★付きで強調表示される（UC-A07：差分プレビューを確認する）
4. 内容を確認し「この内容で適用」を押す
5. 変更件数が50件以上の場合、確認ダイアログが追加で表示される
6. サーバー側でDBを再読込し差分を再計算した上で1トランザクションとして反映し、結果（追加／変更／削除／リストアそれぞれの件数）がトーストで表示される

**例外処理**
- ファイル内にユーザー名が空の行がある、またはユーザー名が重複している行がある → 適用時に検証エラーとなり、エラー内容が画面に表示される（プレビュー自体は表示できるが適用はできない）
- ファイルの読み込みに失敗した場合（破損・非対応形式等） → エラートーストが表示される
- 「削除」と判定されるのは、ファイルの「削除フラグ」列がON、または「解約日」列に日付が入っている行のみ。**ファイルに存在しない行が誤って削除されることはない**（安全設計。詳細は`docs/アカウント認証_Excel取り込み設計.md`参照）

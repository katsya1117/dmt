# データ型定義書 — アカウント認証機能

（作成: 2026-07-13。客先MySQLの実スキーマ（IDEで開かれていた`account_auth`テーブル定義）を正典とし、`server/src/db.ts`のSQLiteデモスキーマ、および`client/src/api/accountAuth.ts`のTypeScript型と突き合わせて作成）

## 正典（客先MySQL）

```sql
CREATE TABLE IF NOT EXISTS `account_auth` (
    `id` int(11) NOT NULL AUTO_INCREMENT,
    `username` varchar(40) CHARACTER SET utf8 NOT NULL,
    `password` varchar(256) COLLATE utf8_bin NOT NULL,
    `comment` varchar(2000) COLLATE utf8_bin DEFAULT NULL,
    `number` int(11) DEFAULT NULL COMMENT 'No.',
    `submission_date` date DEFAULT NULL COMMENT '申込日',
    `regist_date` date DEFAULT NULL COMMENT '登録日',
    `company_cd` varchar(10) CHARACTER SET utf8 DEFAULT NULL COMMENT '販社CD',
    `company_name` varchar(400) CHARACTER SET utf8 DEFAULT NULL COMMENT '販売会社',
    `company_store_cd` varchar(10) CHARACTER SET utf8 DEFAULT NULL COMMENT '販売会社店舗CD',
    `company_store_branch_num` varchar(10) CHARACTER SET utf8 DEFAULT NULL COMMENT '販売会社店舗CD枝番',
    `non_sync` tinyint(1) NOT NULL DEFAULT '0' COMMENT '診断データ対象外アカウント',
    `store_cd` varchar(10) CHARACTER SET utf8 DEFAULT NULL COMMENT '販売店CD',
    `store_name` varchar(400) CHARACTER SET utf8 DEFAULT NULL COMMENT '販売店名',
    `reg_date` datetime NOT NULL,
    `upd_date` timestamp NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    `delfg` tinyint(1) NOT NULL DEFAULT '0' COMMENT '削除フラグ',
    PRIMARY KEY (`id`),
    UNIQUE KEY `username` (`username`),
    KEY `number` (`number`)
) ENGINE=MyISAM DEFAULT CHARSET=utf8 COLLATE=utf8_bin;
```

## フィールド定義

### 基本項目

| 項目名 | MySQL型 | NULL | デフォルト | 説明 |
|---|---|---|---|---|
| id | int(11) | NOT NULL | AUTO_INCREMENT | 主キー |
| username | varchar(40) | NOT NULL | - | ユーザー名。一意制約あり |
| password | varchar(256) | NOT NULL | - | パスワード（平文で格納。ハッシュ化はされていない） |
| comment | varchar(2000) | NULL可 | NULL | 備考 |
| number | int(11) | NULL可 | NULL | No.（インデックスあり） |
| submission_date | date | NULL可 | NULL | 申込日 |
| regist_date | date | NULL可 | NULL | 登録日 |

### 会社・店舗情報

| 項目名 | MySQL型 | NULL | 説明 |
|---|---|---|---|
| company_cd | varchar(10) | NULL可 | 販社CD |
| company_name | varchar(400) | NULL可 | 販売会社 |
| company_store_cd | varchar(10) | NULL可 | 販売会社店舗CD |
| company_store_branch_num | varchar(10) | NULL可 | 販売会社店舗CD枝番 |
| store_cd | varchar(10) | NULL可 | 販売店CD |
| store_name | varchar(400) | NULL可 | 販売店名 |

### システム管理項目

| 項目名 | MySQL型 | NULL | デフォルト | 説明 |
|---|---|---|---|---|
| non_sync | tinyint(1) | NOT NULL | 0 | 診断データ対象外アカウントフラグ |
| reg_date | datetime | NOT NULL | - | 登録日時（アプリ側で明示的にセット） |
| upd_date | timestamp | NOT NULL | CURRENT_TIMESTAMP（更新時も自動更新） | 更新日時 |
| delfg | tinyint(1) | NOT NULL | 0 | 削除フラグ（論理削除） |

## 索引・制約

| 種別 | 対象 | 補足 |
|---|---|---|
| PRIMARY KEY | id | |
| UNIQUE KEY | username | アプリ側でも重複時に409を返す（`server/src/repositories/accountAuth.ts`のcreate） |
| KEY（非一意） | number | |

## TypeScript側の型との対応

`AccountAuth`（読み取り型）・`AccountAuthInput`（書き込み型）は`client/src/api/accountAuth.ts`にてOpenAPI仕様から自動生成される（`api/generated/schema.ts`）。対応関係:

| MySQL | TypeScript | 変換 |
|---|---|---|
| tinyint(1)（`non_sync`, `delfg`） | `boolean` | サーバーのrepository層で 0/1 ⇔ true/false を変換（`server/src/repositories/accountAuth.ts`の`toApi()`） |
| `NULL`許容カラム | `string \| null` / `number \| null` | 空文字列は保存時に`null`へ変換（`AccountAuthFormDialog.tsx`の`orNull()`） |
| date / datetime / timestamp | `string` | ISO的な文字列として扱う。日付専用の型は使っていない |

## 現状の課題（要確認・要対応事項）

- **文字数制限の未検証**: MySQL側は`varchar(40)`（username）や`varchar(10)`（各CD系）など桁数制限があるが、クライアント側のZodスキーマ（`AccountAuthFormDialog.tsx`）は必須チェックのみで**最大文字数のバリデーションが無い**。本番DB接続時に長すぎる入力でエラーになる可能性がある
- **パスワードは平文管理**: `password`カラムはハッシュ化されておらず平文と思われる（客先の既存システム仕様に準拠していると推測されるが、要確認）
- SQLiteデモ（`server/src/db.ts`）は上記MySQLスキーマを型のみ読み替えて再現しているが、**桁数・ENGINE・COLLATEの制約は反映されていない**（SQLiteの性質上、桁数制限はそもそも強制されない）

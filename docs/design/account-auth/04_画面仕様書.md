# 画面仕様書 — アカウント認証機能

（作成: 2026-07-12。`03_画面遷移図.md`に登場する3画面/ダイアログの詳細仕様。実装から逆引き）

## 1. アカウント認証一覧画面（`AccountAuthTable.tsx`）

### 目的
登録済みアカウント認証情報を一覧表示し、新規追加・編集・Excel一括反映の起点となる。

### 項目・部品

| 種別 | 名称 | 内容 |
|---|---|---|
| ボタン | 新規追加 | UC-A02を起動 |
| ボタン | Excel取り込み（差分プレビュー） | ファイル選択→UC-A06を起動 |
| 入力欄 | No./ユーザー名で絞り込み | 入力のたびに一覧をクライアント側でリアルタイム絞り込み（`onChange`） |
| テーブル列 | ID, ユーザー名, パスワード, No., 申込日, 登録日, 販社CD, 販売会社, 販売会社店舗CD, 枝番, 販売店CD, 販売店名, 備考, 登録日時, 更新日時 | `account_auth`テーブルの全カラムをほぼそのまま表示（`id`除き`AccountAuthInput`と対応） |
| テーブル列 | 診断対象外 | `non_sync`を「対象外」（警告色）／「通常」チップで表示 |
| テーブル列 | 状態 | `delfg`を「削除済み」（赤）／「有効」（緑・アウトライン）チップで表示。削除済み行は行全体を半透明表示にする |
| 操作列 | 編集アイコン | UC-A03を起動 |

### 状態・挙動

| 状態 | 表示 |
|---|---|
| 読み込み中 | 「読み込み中...」の行を表示 |
| データが0件 | 「データがありません」の行を表示 |
| 絞り込み結果が0件 | 「該当する行がありません」の行を表示 |
| 通信エラー | 一覧上部にエラーアラートを表示 |
| 操作成功/失敗 | 画面下部にトースト（成功=緑／失敗=赤）を3秒間表示 |

### 削除機能について（実装上の制約）
一覧上に削除ボタンは無い。物理削除API（`DELETE /api/account-auth/:id`）はコード上コメントアウトされ無効化されている（客先DBで物理削除が不調なため）。削除は必ず編集ダイアログの削除フラグ経由（UC-A04）。

## 2. 新規追加／編集ダイアログ（`AccountAuthFormDialog.tsx`）

### 目的
UC-A02（新規追加）・UC-A03〜A05（編集／削除／リストア）を1つのフォームで扱う。

### 入力項目（表示順）

| 項目名 | ラベル | 型 | 必須 | 備考 |
|---|---|---|---|---|
| username | ユーザー名 | 文字列 | ✅ | 一意制約あり（重複時409） |
| password | パスワード | 文字列 | ✅ | |
| number | No. | 数値 | - | 空欄可（nullになる） |
| non_sync | 診断データ対象外 | スイッチ | - | |
| submission_date | 申込日 | 日付 | - | |
| regist_date | 登録日 | 日付 | - | |
| company_cd | 販社CD | 文字列 | - | |
| company_name | 販売会社 | 文字列 | - | |
| company_store_cd | 販売会社店舗CD | 文字列 | - | |
| company_store_branch_num | 店舗CD枝番 | 文字列 | - | |
| store_cd | 販売店CD | 文字列 | - | |
| store_name | 販売店名 | 文字列 | - | |
| comment | 備考 | 複数行文字列 | - | |
| delfg | 削除フラグ（ONで論理削除。一覧には残り「状態」列が削除済みになります） | スイッチ（危険色） | - | ONでUC-A04相当、OFFでUC-A05相当 |

### ボタン

| ボタン | 挙動 |
|---|---|
| キャンセル | 何も送信せずダイアログを閉じる |
| 追加（新規時）／更新（編集時） | 送信中は無効化。成功でダイアログが閉じトースト表示、失敗時は閉じずにエラー表示 |

### バリデーション・エラー表示
- クライアント側: ユーザー名・パスワードの未入力を送信前に検知し、当該項目の下に赤字表示（`react-hook-form`の`mode: 'onTouched'`）
- サーバー側: ユーザー名重複（409）→ユーザー名欄に赤字表示。それ以外のエラー→ダイアログ上部に汎用エラーアラート

## 3. 差分プレビューダイアログ（`ImportDiffDialog.tsx`）

### 目的
UC-A07。Excel取り込み内容とDBの差分を、承認前にテーブル形式で確認する。

### 項目・部品

| 種別 | 名称 | 内容 |
|---|---|---|
| 注意書き | 案内アラート | 「この画面ではまだDBに書き込みません。内容を確認してください（★の付いた認証系の変更は特に注意）。」 |
| テーブル列 | 区分 | 追加（緑）／変更（青）／削除（赤）／リストア（オレンジ）のチップ |
| テーブル列 | ユーザー名, パスワード, No., 申込日, 登録日, 販社CD, 販売会社, 販売会社店舗CD, 枝番, 販売店CD, 販売店名, 備考 | 一覧画面と同じ項目セット（変更セルは太字＋色で強調。ユーザー名/パスワード/削除フラグの変更は★付きでさらに強調） |
| キャプション | 変更なし件数 | 「変更なし：n件（表示しません）」 |

### ボタン

| ボタン | 挙動 |
|---|---|
| 閉じる | ダイアログを閉じる（DB未反映のまま） |
| この内容で適用 | UC-A06の適用処理を実行。差分が0件の場合は無効化。変更件数が50件以上の場合はブラウザの確認ダイアログ（`confirm()`）を追加表示。適用中はボタン文言が「適用中…」に変わり無効化される |

### 状態・挙動
- 差分が0件（全区分で0件） → テーブルに「変更はありません」の行を表示し、適用ボタンは無効化

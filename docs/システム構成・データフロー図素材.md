## 1. システム構成図

### ボックス一覧（Excelの「図形」に対応）

| # | ボックス名 | 種別 | 備考 |
|---|-----------|------|------|
| A | ブラウザ（Chrome等） | クライアント | 運用PC上で開く |
| B | React アプリ | フロントエンド | Vite + TypeScript。ポート5173（開発時） |
| C | Express サーバー | バックエンド | Node.js。運用PCで常駐。ポート3001 |
| D | Sambaフォルダ | ストレージ | 運用PC or ファイルサーバー上のマウント済みフォルダ ★実パス要確認 |
| E | 顧客PHPサーバー | 外部サーバー | ★ホスト名・URLは実環境に合わせる |
| F | 型式別フォルダ群 | ストレージ | PHPサーバー上。型式コードごとのサブフォルダ |

### 接続一覧（Excelの「矢印」に対応）

| 矢印 | From | To | プロトコル / 内容 | 方向 |
|------|------|----|-----------------|------|
| ① | A（ブラウザ） | B（React） | 静的ファイル配信（HTML/JS/CSS） | → |
| ② | B（React） | C（Express） | HTTP REST（localhost:3001） | ↔ |
| ③ | C（Express） | D（Sambaフォルダ） | ローカルファイルI/O（fs.readdir / fs.createReadStream） | ↔ |
| ④ | C（Express） | E（PHPサーバー） | HTTP multipart POST（node-fetch） | → |
| ⑤ | E（PHPサーバー） | F（型式別フォルダ） | ファイル書き込み | → |

## 2. データフロー図（アップロード操作）

### フロー説明（シーケンス）

#### フェーズ1: 型式一覧の取得

| ステップ | 処理主体 | 内容 |
|---------|---------|------|
| 1 | ブラウザ（React） | `GET /api/katashiki` を送信 |
| 2 | Express | Sambaフォルダのサブディレクトリ名を列挙して返す |
| 3 | ブラウザ（React） | 型式リストを画面に表示（約700件） |

#### フェーズ2: 対象ファイルの確認

| ステップ | 処理主体 | 内容 |
|---------|---------|------|
| 4 | ブラウザ（React） | 型式を選択 → `GET /api/katashiki/{id}/files` を送信 |
| 5 | Express | Sambaの型式フォルダ内ファイルを列挙（拡張子でフィルタ） |
| 6 | ブラウザ（React） | ファイル一覧・サイズを表示。型式別DB（.db）の有無を確認 |

#### フェーズ3: アップロード実行

| ステップ | 処理主体 | 内容 |
|---------|---------|------|
| 7 | ブラウザ（React） | 「アップロード」ボタン押下 → `POST /api/upload/{id}` を送信 |
| 8 | Express | Sambaフォルダから対象ファイルをストリームで読み込む |
| 9 | Express | 読み込んだファイルを multipart/form-data に詰め直す |
| 10 | Express | `POST {PHPサーバーURL}/api/upload.php` へ転送 |
| 11 | PHPサーバー | 受け取ったファイルを型式別フォルダに保存 |
| 12 | Express | PHPのレスポンス（成否）をブラウザへ返す |
| 13 | ブラウザ（React） | 成功/失敗を進捗UIに反映。失敗分は再実行ボタンを表示 |

## 3. Excelでの作り方ヒント

### シート構成案

| シート名 | 内容 |
|---------|------|
| システム構成図 | ボックスと矢印で構成図を作成 |
| データフロー | 水泳レーン（スイムレーン）形式でシーケンスを表現 |

### 図形の使い方

| 要素 | Excelの図形 | 塗り色の目安 |
|------|------------|-------------|
| ブラウザ / React | 角丸四角形 | 水色 |
| Express（バックエンド） | 四角形 | オレンジ |
| Samba / ファイルストレージ | 平行四辺形 or シリンダー | グレー |
| PHPサーバー（外部） | 四角形（太枠） | 薄緑 |
| 型式別フォルダ | シリンダー | 薄緑 |
| 矢印 | コネクタ（肘線） | 黒 |
| ラベル（HTTP等） | テキストボックス | 枠なし |

### スイムレーンの作り方（データフロー図）

1. 列A〜Dを広めに取り、それぞれ「ブラウザ」「Express」「Samba」「PHPサーバー」とヘッダーをつける
2. 行ごとに1ステップを表現
3. 矢印は「挿入 → 図形 → 線 → コネクタ（肘線矢印）」で列をまたいで引く
4. 矢印上にラベルを追加するには右クリック→「テキストの追加」

## 4. 確認が必要な項目（★）

作図前に確認しておくと精度が上がります。

| # | 項目 | 現在の仮値 | 確認先 |
|---|------|-----------|--------|
| 1 | Sambaフォルダの実パス | `../mock/samba`（開発用） | 運用担当者 |
| 2 | PHPサーバーのURL / ホスト名 | `http://localhost:8080`（開発用） | 顧客側担当者 |
| 3 | 型式別フォルダの保存先パス | PHPサーバー上 `/var/www/html/uploads/{型式}/` | 顧客側担当者 |
| 4 | 運用PCのOS / Express起動方法 | Windows想定（未確定） | 運用担当者 |
| 5 | ブラウザのポート（本番） | 5173（開発）→ ビルド後は3001同居？ | 要設計 |

# セットアップ手順（コードをpullして動かす）

ラボで開発したコードを、業務環境（運用PC）でpullして動かす際の手順。
（最終更新: 2026-06-25）

## 前提

- **Node.js 22 系**（ラボは v22.14。メジャーを揃えること。ネイティブモジュールの都合で重要）
- パッケージ管理：ルートと client は **yarn**、server は **npm**（既存のlockfileに合わせる）

## 1. 依存インストール

3か所でインストールが必要。

```
# リポジトリのルートで
yarn install                 # ルート（concurrently）

cd client && yarn install    # フロント
cd ../server && npm install  # バックエンド（better-sqlite3 を含む）
```

### ⚠️ 重要：better-sqlite3 はネイティブモジュール

server の `npm install` で **better-sqlite3**（v12系）をビルド／取得する。
通常は対応プラットフォームのプリビルドバイナリが自動で落ちてくるが、
**Windows で該当バイナリが無い場合はソースからのビルドが走り、ビルドツールが必要**になる。

その場合に必要なもの（Windows）:
- Visual Studio Build Tools（C++ ワークロード）
- Python 3.x

ビルドで失敗する場合の典型対処:
- Node のメジャーバージョンをラボと揃える（プリビルドが見つかりやすくなる）
- 上記ビルドツールを入れて `npm install` を再実行

> 将来、ネイティブビルドを避けたい場合は Node 標準の `node:sqlite` への移行も検討可（ただし現状は better-sqlite3 を採用）。

## 2. 環境変数（server）

`server/.env` は gitignore 済み。`.env.example` をコピーして用意する。

```
cd server
cp .env.example .env
```

未設定でも `config.ts` の既定値（PORT=3001 / SAMBA_PATH=../mock/samba / PHP_API_URL=http://localhost:8080）で動くが、
本番環境では実際のパス・URLに合わせて `.env` を編集する。

## 3. 起動

ルートで1コマンド：

```
yarn dev      # client(5173) と server(3001) を同時起動
```

個別起動：`yarn dev:client` / `yarn dev:server`

## 4. DB（SQLite）について

### 手動セットアップは不要

server 起動時に **`server/data/demo.db` が自動生成**される（`server/src/db.ts`）。
- データディレクトリ・DBファイル・テーブルを自動作成（CREATE TABLE IF NOT EXISTS）
- 空のときだけデモ用データを自動投入（seed）

→ **pull 後は依存インストールして起動するだけ**でDBは用意される。

### 注意：これはデモ用のローカルDB（B3方式）

- 本番の顧客DBではなく、開発・デモ用の代替（`docs/デプロイ構成.md` 参照）
- スキーマは**仮**（既存AMFPHPの実テーブルに後で合わせ込む）
- 本番化時は Express の `repositories/` を PHP(REST)呼び出しに差し替える設計

### DBのリセット

ファイルを消すだけ。次回起動で再作成＋再seedされる。

```
rm server/data/demo.db server/data/demo.db-wal server/data/demo.db-shm
```

### DBの中身を見る

```
# CLI（macに標準のsqlite3）
sqlite3 -header -column server/data/demo.db "SELECT * FROM account_auth;"
```

GUIで見るなら DB Browser for SQLite（`brew install --cask db-browser-for-sqlite`）等で
`server/data/demo.db` を開く。

## まとめ（最短手順）

```
yarn install
cd client && yarn install && cd ..
cd server && npm install && cp .env.example .env && cd ..
yarn dev
```

DBは初回起動で自動作成される。Windowsで better-sqlite3 のビルドに失敗したら、
Nodeバージョンを揃える／ビルドツールを入れる。

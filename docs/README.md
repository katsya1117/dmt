## 決定の変遷（方針転換ログ）

検討の過程で方針が変わった項目。**右が最新（採用）**。古い前提が他資料に残っていたら最新に従う。

| 項目 | 当初 | → 最新（採用） | 理由 |
|------|------|----------------|------|
| UIライブラリ | Tailwind + MUI 併用 | **MUI 単独**（Tailwind撤去） | 共存の摩擦・シンプル優先 |
| ナビゲーション | サイドバー型 | **ヘッダーのホバードロップダウン**（親6メニュー） | メニュー数増加に対応 |
| 画面構成 | 4画面（S1〜S4） | **6親メニュー / 27画面** | 機能の精緻化 |
| 認証・権限 | 認証なし・ログイン不要 | **認証・権限管理が中核** | 客先で**顧客社員**が一部機能を使うため |
| ハード構成 | 社内1台に集約 | **社内Windows1台 ＋ 客先軽量デプロイの2拠点** | 客先利用の判明 |
| デモのデータ源 | （未定） | **B3方式: Express + SQLite**（本番は `repositories/` 差し替え） | 客先DBに未接続のため |
| Excel解析 | xlsx (SheetJS) | **exceljs** | xlsxはnpm版が塩漬け＋CVE |
| React | 18（安定重視で据置） | **19**（依存全面更新） | 全面更新を実施 |
| HTTP通信 | node-fetch | **Node組み込み fetch** | 不要依存の削除 |
| API層 | 手書きExpressルート＋手書き検証 | **tsoa**（型からルート/検証/OpenAPI生成）＋Swagger UI | 型の単一の真実・検証自動化 |
| client/serverの型 | 各々で二重定義 | **OpenAPI仕様から自動生成**（openapi-typescript） | 二重管理の解消 |
| クライアントHTTP通信 | 組み込み fetch | **axios** | 共通インスタンス・エラー変換の一元化 |
| 一覧テーブル | 手組みMUI Table | **MUI X DataGrid**（Community・無料） | 大量データ（実測20000行）でのブラウザ固まり対策。仮想化標準搭載 |

## 確定している中核方針

- **ReactはExpressとだけ通信**する（PHP・処理A/B・DBを直接意識しない）
- **本番化の差し替え点は `server/src/repositories/`**（SQLite→客先DB/PHP）。画面・フック・API・コントローラは無変更
- **API契約は tsoa コントローラの型が源泉**（→OpenAPI仕様→クライアント型を生成）。将来PHPが満たす契約書にもなる
- **状態管理**: サーバー状態=TanStack Query / クライアント状態=Redux Toolkit
- **テスタビリティ**: APIはフックに閉じる・MSWでモック・実装とStorybookはセット

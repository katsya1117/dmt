# 依存アップデート 簡易版（3分）

ふだんの軽い更新用。メジャー跨ぎや大掛かりな更新は → `依存アップデート手順.md`（詳細版）。

---

## 鉄則（3つだけ）

1. **`--latest` を引数なしで使わない**（全部メジャー更新＝事故）
2. **更新したら型・テスト・ビルドを通す**
3. **1更新 = 1コミット**（壊れたら revert で戻す）

---

## 手順

```bash
# 0. クリーンを確認（汚れてたら先にコミット/退避）
git status

# 1. 何が遅れてる？
npm outdated          # yarn: yarn outdated

# 2. 安全な範囲内だけ上げる（^の中。メジャーは動かない）
npm update            # yarn: yarn upgrade

# 3. 検証（1つでも赤なら止める）
npx tsc --noEmit      # 型
npx vitest run        # テスト
npm run build         # ビルド

# 4. OKならコミット
git add -A && git commit -m "deps: 範囲内更新"
```

これで「安全枠」は完了。**メジャー（Wanted < Latest）は簡易版では触らない** → 詳細版へ。

---

## 脆弱性をサッと見る

```bash
npm audit --audit-level=high    # exit 0 なら high以上なし＝OK
# yarn(v1): yarn audit --level high
```

`npm audit fix` は OK。**`--force` は使わない**（依存を壊すことがある）。

## 困ったら

```bash
git checkout -- .     # 未コミットの更新を全部捨てる
npm install           # lockに合わせて戻す（yarn: yarn install）
```
